---
title: UI Pages
description: Server-rendered authentication pages for sign-in, sign-up, password reset, and more
---

Better Auth provides built-in UI pages for common authentication flows. You can use these pages in two ways:

1. **Full pages** - Access directly in the browser (e.g., `/auth/sign-in`)
2. **Embedded components** - Render inside your app using the `<Auth />` component (see framework integration docs)

## Enabling UI Pages

To enable UI pages, add the `ui` option to your Better Auth configuration:

```ts title="auth.ts"
import { betterAuth } from "better-auth";

export const auth = betterAuth({
  // ... other options
  ui: {
    enabled: true,
  },
});
```

## Mounting the UI Handler

The UI pages are served through a separate handler (`uiHandler`) that you mount in your app:

```ts title="server.ts"
import { auth } from "./auth";

// Mount the UI handler - handles routes under /auth/*
app.use("/*", (req) => auth.uiHandler(req));

// Mount the API handler for auth API endpoints
app.use("/api/auth/*", (req) => auth.handler(req));
```

Your auth pages will be available at:

* `/auth/sign-in` - Sign in page
* `/auth/sign-up` - Sign up page
* `/auth/forgot-password` - Forgot password page
* `/auth/reset-password` - Reset password page
* `/auth/verify-email` - Email verification page
* `/auth/profile` - User profile page

## Theming

### Automatic Theme Detection

When using the `<Auth />` component, shadcn CSS variables from your parent page are automatically detected and applied to the iframe. No manual configuration needed if your app already uses shadcn.

### Custom CSS URL

For zero-flash theming, provide a CSS file URL:

```ts title="auth.ts"
export const auth = betterAuth({
  ui: {
    cssURL: "/themes/my-shadcn-theme.css", // Loaded before React renders
  },
});
```

The CSS file should contain shadcn CSS variables:

```css title="public/themes/my-shadcn-theme.css"
:root {
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  --primary: 222.2 47.4% 11.2%;
  --primary-foreground: 210 40% 98%;
  /* ... other shadcn variables */
}
```

## Page Metadata

Customize SEO and social sharing metadata:

```ts title="auth.ts"
export const auth = betterAuth({
  ui: {
    metadata: ({ page, path }) => ({
      title: `${page} - My App`,
      description: "Secure authentication for My App",
      image: "https://myapp.com/og-auth.png",
      twitterCard: "summary_large_image",
    }),
  },
});
```

## Page Configuration

Enable or disable individual pages:

```ts title="auth.ts"
export const auth = betterAuth({
  ui: {
    pages: {
      signIn: { enabled: true },
      signUp: { enabled: true },
      forgotPassword: { enabled: true },
      resetPassword: { enabled: true },
      verifyEmail: { enabled: true },
      profile: { enabled: false },
    },
  },
});
```

## Local Development

When developing with different ports (e.g., Next.js on :3000, API on :4000), set up a proxy:

### Next.js

```js title="next.config.js"
module.exports = {
  async rewrites() {
    return [
      { source: '/auth/:path*', destination: 'http://localhost:4000/auth/:path*' }
    ];
  },
};
```

### Vite

```ts title="vite.config.ts"
export default defineConfig({
  server: {
    proxy: {
      '/auth': 'http://localhost:4000'
    }
  }
});
```

## Plugin UI Pages

Plugins can expose additional UI pages. When a plugin provides UI pages, they become available through `authClient.ui`:

```tsx
// If a plugin like email-otp provides a UI page
<Auth ui={authClient.ui.emailOtp({ email: "user@example.com" })} />
```

### Creating Plugin UI Pages

Plugins can define UI pages in both server and client configurations:

```ts title="Server plugin"
export const myPlugin = () => ({
  id: "my-plugin",
  ui: {
    pages: [{ 
      path: "/my-page", 
      html: myPageHTML,
      title: "My Plugin Page"
    }]
  },
});
```

```ts title="Client plugin"
export const myPluginClient = () => ({
  id: "my-plugin",
  ui: {
    myPage: {
      path: "/my-page",
      args: {} as { customArg?: string }
    }
  }
});
```

The client plugin's `args` type enables TypeScript autocomplete when using `authClient.ui.myPage({ customArg: "..." })`.

## Full Configuration Reference

```ts
interface UIOptions {
  enabled?: boolean;
  
  pages?: {
    signIn?: { enabled?: boolean };
    signUp?: { enabled?: boolean };
    forgotPassword?: { enabled?: boolean };
    resetPassword?: { enabled?: boolean };
    verifyEmail?: { enabled?: boolean };
    profile?: { enabled?: boolean };
  };
  
  basePath?: string;
  apiBaseUrl?: string;
  redirectTo?: string;
  
  /** URL to custom CSS file for theming */
  cssURL?: string;
  
  /** Callback to customize page metadata */
  metadata?: (context: { page: string; path: string }) => {
    title?: string;
    description?: string;
    image?: string;
    ogType?: string;
    twitterCard?: "summary" | "summary_large_image";
    additionalMeta?: Array<{ name?: string; property?: string; content: string }>;
  };
}
```
