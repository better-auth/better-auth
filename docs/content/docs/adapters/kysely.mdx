Kysely is a type-safe SQL query builder for TypeScript that supports multiple relational databases such as PostgreSQL, MySQL, SQLite, and more. It provides a flexible and strongly-typed way to build and execute SQL queries.

Before getting started, make sure you have a supported database set up (such as PostgreSQL, MySQL, or SQLite) and Kysely configured in your project. For more information, see the [Kysely Documentation](https://kysely.dev/docs/intro).

## Installation

Kysely support is built into Better Auth. You only need to install Kysely:

```package-install
kysely
```

## Example Usage

For this example, you will also need:

```bash
npm install kysely-planetscale @planetscale/database
```

You can use the Kysely adapter to connect to your database as follows.

```ts title="auth.ts"
import { betterAuth } from "better-auth";
import { Kysely } from "kysely";
import { PlanetScaleDialect } from "kysely-planetscale";
import { connect } from "@planetscale/database";

const db = new Kysely({
  dialect: new PlanetScaleDialect({
    database: connect({
      url: process.env.DATABASE_URL!,
    }),
  }),
});

export const auth = betterAuth({
  database: db,
});
```

## Schema generation & migration

Better Auth CLI supports schema generation and migrations when using Kysely:

```bash
npx auth@latest generate
npx auth@latest migrate
```

## Joins (Experimental)

Database joins are useful when Better-Auth needs to fetch related data from multiple tables in a single query.
Endpoints like `/get-session`, `/get-full-organization` and many others benefit greatly from this feature,
seeing upwards of 2x to 3x performance improvements depending on database latency.

The Kysely adapter supports joins through Better Auth’s experimental joins feature.
To enable this, set the `experimental.joins` option to `true` in your auth configuration.

```ts title="auth.ts"
import { betterAuth } from "better-auth";

export const auth = betterAuth({
  experimental: { joins: true }
});
```
