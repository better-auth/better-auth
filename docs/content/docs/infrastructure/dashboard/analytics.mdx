## API Endpoints

Access analytics data programmatically:

### Get User Statistics

```ts
const stats = await authClient.dash.getUserStats({
  session: sessionData,
});
```

### Get Graph Data

```ts
const graphData = await authClient.dash.getUserGraphData({
  session: sessionData,
  period: "30d",  // 7d, 30d, 90d, 12m
});
```

### Get Retention Data

```ts
const retention = await authClient.dash.getUserRetentionData({
  session: sessionData,
});
```

### Get Geographic Data

```ts
const mapData = await authClient.dash.getUserMapData({
  session: sessionData,
});
```

### Get Online Users Count

```ts
const online = await authClient.dash.getOnlineUsersCount({
  session: sessionData,
});
```

## Best Practices

1. **Regular monitoring** - Check analytics weekly to spot trends early.

2. **Set benchmarks** - Establish baseline metrics to measure against.

3. **Investigate anomalies** - Sudden changes may indicate issues or opportunities.

4. **Compare periods** - Use period-over-period comparison for context.

5. **Export for reporting** - Generate regular reports for stakeholders.

## Interpreting Metrics

### Healthy Signs

- Steady or growing user count
- Good retention rates (>40% Day 7)
- Consistent daily active users
- Low failed login rates

### Warning Signs

- Declining signups
- Dropping retention
- Unusual geographic patterns
- Spike in failed authentications

### Taking Action

- Low retention → Improve onboarding
- Geographic concentration → Consider localization
- Device gaps → Test on underperforming platforms
- Method preferences → Promote popular auth methods
