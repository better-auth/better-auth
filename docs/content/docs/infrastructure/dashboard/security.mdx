## Overview

## Event Types

### Credential Stuffing

Detected when multiple failed login attempts occur from the same visitor.

**Event data includes:**
- Visitor ID
- Number of attempts
- Target identifiers (emails)
- Password hash patterns (for same-password detection)

**Indicators:**
- Multiple failed logins in short period
- Same password tried across accounts
- Automated request patterns

### Impossible Travel

Detected when a user logs in from distant locations in an impossibly short time.

**Event data includes:**
- User ID
- From location (city, country)
- To location (city, country)
- Distance (km)
- Time elapsed (hours)
- Required speed (km/h)

**Example:**
> User logged in from New York, then Tokyo 30 minutes later (would require 25,000 km/h travel)

### Geo-Blocking

Triggered when requests come from blocked countries.

**Event data includes:**
- Country code
- Country name
- IP address
- Blocked reason (deny list or not in allow list)

### Bot Detection

Triggered when automated bot traffic is detected.

**Event data includes:**
- Detection signals
- User agent analysis
- Behavioral patterns

### Suspicious IP

Triggered for known malicious IP addresses.

**Event data includes:**
- IP address
- Reputation source
- Threat category

### Velocity Exceeded

Triggered when rate limits are exceeded.

**Event data includes:**
- Rate type (signups, logins, password resets)
- Current count
- Threshold
- Window period

### Free Trial Abuse

Detected when the same device creates multiple accounts.

**Event data includes:**
- Visitor fingerprint
- Account count
- Max allowed
- User IDs created

### Compromised Password

Detected when a user tries to set a password found in data breaches.

**Event data includes:**
- Breach count (how many breaches contain this password)
- Action taken

### Stale Account Reactivation

Detected when a dormant account suddenly becomes active.

**Event data includes:**
- User ID
- Days since last activity
- Stale threshold (days)
- Last active date
- Notification sent (user/admin)

## Alert Indicators

The dashboard highlights concerning patterns:

### High Severity

- Sudden spike in blocked events
- New geographic attack source
- Credential stuffing campaign detected

### Medium Severity

- Elevated failed login rate
- Unusual traffic patterns
- Multiple impossible travel events

### Low Severity

- Individual blocked requests
- Single compromised password attempt
- Minor rate limit triggers

## Best Practices

1. **Regular review** - Check security dashboard daily for new threats.

2. **Investigate patterns** - Look for coordinated attacks across IPs.

3. **Tune thresholds** - Adjust Sentinel settings based on observed patterns.

4. **Document incidents** - Keep records of significant security events.

5. **Set up alerts** - Configure notifications for critical events (Enterprise).

## Responding to Threats

### Credential Stuffing Campaign

1. Review targeted accounts
2. Force password reset if compromised
3. Consider additional authentication requirements
4. Block persistent attackers

### Geographic Attack Surge

1. Review geo-blocking settings
2. Consider expanding deny list
3. Investigate targeted resources

### Bot Attack

1. Verify bot detection is enabled
2. Consider increasing challenge difficulty
3. Review rate limits

### Compromised Credentials

1. Notify affected users
2. Force password reset
3. Review password policies
