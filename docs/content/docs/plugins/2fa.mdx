`OTP` `TOTP` `Recovery Codes` `Trusted Devices`

Two-Factor Authentication (2FA) adds an extra verification step when users log in. This plugin supports two main verification methods:

1. **OTP (One-Time Password)**: A temporary code sent to the user by your `sendOTP` implementation.
2. **TOTP (Time-Based One-Time Password)**: A code generated by an authenticator app on the user's device.

**Additional features include:**

* Recovery codes for account recovery
* Trusted devices
* Enabling, disabling, listing, and removing 2FA methods

## Installation

<Steps>
  <Step>
    ### Add the plugin to your auth config

    Add the `twoFactor` plugin to your auth config and, if needed, provide your app name as the issuer for `TOTP`.

    ```ts title="auth.ts"
    import { betterAuth } from "better-auth"
    import { twoFactor } from "better-auth/plugins" // [!code highlight]

    export const auth = betterAuth({
        appName: "My App", // Used as the default TOTP issuer // [!code highlight]
        plugins: [
            twoFactor() // [!code highlight]
        ]
    })
    ```
  </Step>

  <Step>
    ### Migrate the database

    Run the migration or generate the schema to add the required tables.

    <Tabs items={["migrate", "generate"]}>
      <Tab value="migrate">
        ```package-install
        npx auth migrate
        ```
      </Tab>

      <Tab value="generate">
        ```package-install
        npx auth generate
        ```
      </Tab>
    </Tabs>

    See the [Schema](#schema) section to add the tables manually.
  </Step>

  <Step>
    ### Add the client plugin

    ```ts title="auth-client.ts"
    import { createAuthClient } from "better-auth/client"
    import { twoFactorClient } from "better-auth/client/plugins" // [!code highlight]

    export const authClient = createAuthClient({
        plugins: [
            twoFactorClient() // [!code highlight]
        ]
    })
    ```
  </Step>
</Steps>

## Usage

### Enable 2FA

Start by enabling either `TOTP` or `OTP` for the current signed-in user. Better Auth creates recovery codes the first time the user enables a 2FA method.

#### Enable TOTP

Use `enableTotp()` to create a `TOTP` method and receive the provisioning URI for the authenticator app.

<APIMethod path="/two-factor/enable-totp" method="POST" requireSession>
  ```ts
  type enableTotp = {
      /**
       * The user's password if the account has a credential account.
       */
      password?: string = "secure-password"
      /**
       * Optional label for this TOTP method.
       */
      label?: string = "Authenticator app"
      /**
       * Optional custom issuer for the TOTP URI.
       */
      issuer?: string = "my-app-name"
  }
  ```
</APIMethod>

`enableTotp()` returns:

* `method`: the created `TOTP` method
* `totpURI`: the URI to display as a QR code
* `recoveryCodes`: recovery codes, when they are created for the first time

```ts
const { data } = await authClient.twoFactor.enableTotp({
    password: "secure-password", // [!code highlight]
    label: "Authenticator app", // [!code highlight]
})

console.log(data?.method.id)
console.log(data?.totpURI)
console.log(data?.recoveryCodes)
```

If `skipVerificationOnEnable` is disabled, verify the `TOTP` method after the user scans the QR code:

```ts
await authClient.twoFactor.verify({
    methodId: data!.method.id, // [!code highlight]
    code: "123456",
})
```

#### Enable OTP

Use `enableOtp()` to create or update the user's `OTP` method.

<APIMethod path="/two-factor/enable-otp" method="POST" requireSession>
  ```ts
  type enableOtp = {
      /**
       * The user's password if the account has a credential account.
       */
      password?: string = "secure-password"
      /**
       * Optional label for this OTP method.
       */
      label?: string = "Email OTP"
  }
  ```
</APIMethod>

When `skipVerificationOnEnable` is `false` (the default), Better Auth sends an `OTP` immediately so the user can verify the method:

```ts
const { data } = await authClient.twoFactor.enableOtp({
    password: "secure-password", // [!code highlight]
    label: "Email OTP", // [!code highlight]
})

if (data?.codeSent) {
    await authClient.twoFactor.verify({
        methodId: data.method.id, // [!code highlight]
        code: "123456",
    })
}
```

### Disable 2FA

To disable 2FA entirely, call `disable()`. This removes all methods for the user and revokes all trusted devices.

<APIMethod path="/two-factor/disable" method="POST" requireSession>
  ```ts
  type disableTwoFactor = {
      /**
       * The user's password if the account has a credential account.
       */
      password?: string
  }
  ```
</APIMethod>

### Sign In with 2FA

Once a user has a verified `TOTP` or `OTP` method, Better Auth pauses future sign-ins before issuing the session cookie. For fetch-based flows, the response resolves to a challenge envelope:

```ts
{
    kind: "challenge",
    challenge: {
        kind: "two-factor",
        attemptId: string,
        methods: [
            {
                id: "method_123",
                kind: "totp",
                label: "Authenticator app"
            },
            {
                id: "method_456",
                kind: "otp",
                label: "Email OTP"
            },
            {
                id: "method_789",
                kind: "recovery-code",
                label: null
            }
        ]
    }
}
```

A finalized sign-in instead returns `{ kind: "session", session, user }`. Use `challenge.methods` to decide which verification UI to show for the available `TOTP`, `OTP`, or recovery code options, and pass the selected `methodId` to `verify()`.

```tsx
import { authClient } from "@/lib/auth-client"
import { isSignInChallengeOfKind } from "better-auth"

await authClient.signIn.email({
        email: "user@example.com",
        password: "password123",
    },
    {
        async onSuccess(context) {
            if (isSignInChallengeOfKind(context.data, "two-factor")) { // [!code highlight]
                const { attemptId, methods } = context.data.challenge
                const totpMethod = methods.find((method) => method.kind === "totp")

                // Show the appropriate 2FA verification UI
                // and pass `methodId` to authClient.twoFactor.verify()
                console.log(attemptId, totpMethod?.id)
            }
        },
    }
)
```

<Callout>
  No primary session cookie is issued until the user successfully completes `verify()`.
</Callout>

You can also handle paused sign-ins globally with `onTwoFactorRedirect` or `twoFactorPage`:

```ts title="auth-client.ts"
import { createAuthClient } from "better-auth/client"
import { twoFactorClient } from "better-auth/client/plugins"

export const authClient = createAuthClient({
    plugins: [
        twoFactorClient({
            onTwoFactorRedirect({ attemptId, methods }){ // [!code highlight]
                const otpMethod = methods.find((method) => method.kind === "otp")
                console.log(attemptId, otpMethod?.id)
            },
        }),
    ],
})
```

```ts title="auth-client.ts"
import { createAuthClient } from "better-auth/client"
import { twoFactorClient } from "better-auth/client/plugins"

export const authClient = createAuthClient({
    plugins: [
        twoFactorClient({
            twoFactorPage: "/two-factor", // [!code highlight]
        }),
    ],
})
```

<Callout type="warn">
  Redirect-based flows such as OAuth callbacks, magic-link verification, and email-verification auto sign-in redirect back with `challenge=two-factor` in the query string. The `attemptId` is not included in the URL. Better Auth stores it in the signed `better-auth.two_factor_challenge` cookie instead.
</Callout>

## TOTP

### Get the TOTP URI again

Use `getTotpUri()` to display the QR code for an existing `TOTP` method.

<APIMethod path="/two-factor/get-totp-uri" method="POST" requireSession>
  ```ts
  type getTotpUri = {
      /**
       * The TOTP method to read.
       */
      methodId: string
      /**
       * The user's password if the account has a credential account.
       */
      password?: string
  }
  ```
</APIMethod>

**Example: Using React**

```tsx
import { authClient } from "@/lib/auth-client"
import QRCode from "react-qr-code"

async function loadQrCode(methodId: string, password?: string) {
    const { data } = await authClient.twoFactor.getTotpUri({
        methodId,
        password,
    })

    return data?.totpURI
}

export default function TotpQr({ totpURI }: { totpURI: string }){
    return <QRCode value={totpURI} />
}
```

<Callout>
  By default, the `TOTP` issuer uses the app name from your auth config. To change the issuer returned by `getTotpUri()`, configure `issuer` on the plugin. Passing `issuer` to `enableTotp()` only affects the URI returned by that enrollment call.
</Callout>

### Verify a TOTP code

Use `verify()` with the selected `TOTP` method:

<APIMethod path="/two-factor/verify" method="POST" requireHeaders headersComment="Pass the current request headers so Better Auth can read and set the 2FA/session cookies.">
  ```ts
  type verifyTwoFactor = {
      /**
       * The selected method to verify.
       */
      methodId: string
      /**
       * The verification code.
       */
      code: string = "123456"
      /**
       * The paused sign-in attempt to complete.
       */
      attemptId?: string
      /**
       * Mark the current device as trusted.
       */
      trustDevice?: boolean = true
  }
  ```
</APIMethod>

```ts
await authClient.twoFactor.verify({
    methodId: "totp-method-id",
    code: "123456",
    trustDevice: true,
})
```

## OTP

`OTP` is a one-time password delivered by your application. Before using `OTP`, configure `sendOTP` in your Better Auth instance.

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { twoFactor } from "better-auth/plugins"

export const auth = betterAuth({
    plugins: [
        twoFactor({
            otpOptions: {
                async sendOTP({ user, otp }, ctx) {
                    // Send the OTP to the user // [!code highlight]
                },
            },
        }),
    ],
})
```

### Read the pending challenge

Use `pendingChallenge()` to read the paused sign-in and inspect the available `OTP`, `TOTP`, or recovery code options.

<APIMethod path="/two-factor/pending-challenge" method="GET" requireHeaders headersComment="Pass the current request headers so Better Auth can read the signed two-factor cookie.">
  ```ts
  type pendingChallenge = {}
  ```
</APIMethod>

### Send an OTP

Use `sendCode()` with the selected `OTP` method.

<APIMethod path="/two-factor/send-code" method="POST" requireHeaders headersComment="Pass the current request headers so Better Auth can read the pending challenge or session cookies.">
  ```ts
  type sendCode = {
      /**
       * The selected OTP method.
       */
      methodId: string
      /**
       * The paused sign-in attempt to continue.
       */
      attemptId?: string
  }
  ```
</APIMethod>

```ts
const { data: challenge } = await authClient.twoFactor.pendingChallenge()
const otpMethod = challenge?.methods.find((method) => method.kind === "otp")

await authClient.twoFactor.sendCode({
    methodId: otpMethod!.id, // [!code highlight]
})
```

### Verify an OTP

Use `verify()` with the same `OTP` method:

```ts
await authClient.twoFactor.verify({
    methodId: "otp-method-id", // [!code highlight]
    code: "123456",
})
```

## Recovery Codes

Recovery codes let users recover access if they lose access to their `OTP` or `TOTP` device. If you previously called these backup codes in your app, the flow is the same.

Recovery codes are created the first time a user enables a 2FA method. To rotate them, call `regenerateRecoveryCodes()`.

<APIMethod path="/two-factor/regenerate-recovery-codes" method="POST" requireSession>
  ```ts
  type regenerateRecoveryCodes = {
      /**
       * The user's password if the account has a credential account.
       */
      password?: string
  }
  ```
</APIMethod>

```ts
const { data } = await authClient.twoFactor.regenerateRecoveryCodes({
    password: "secure-password",
})

console.log(data?.recoveryCodes)
```

<Callout type="warn">
  Regenerating recovery codes replaces all existing unused recovery codes.
</Callout>

### Use a recovery code

Use `verify()` with the recovery code option:

```ts
const { data: challenge } = await authClient.twoFactor.pendingChallenge()
const recoveryMethod = challenge?.methods.find(
    (method) => method.kind === "recovery-code"
)

await authClient.twoFactor.verify({
    methodId: recoveryMethod!.id, // [!code highlight]
    code: "RECOVERY-CODE",
})
```

<Callout>
  Recovery codes are one-time use. After a recovery code is used, it cannot be used again.
</Callout>

## Trusted Devices

You can mark a device as trusted by passing `trustDevice: true` to `verify()`.

```ts
await authClient.twoFactor.verify({
    methodId: "totp-method-id",
    code: "123456",
    trustDevice: true, // [!code highlight]
})
```

When `trustDevice` is set to `true`, the current device is remembered for the configured trust window. During that time, the user is not prompted for 2FA on subsequent sign-ins from the same device.

## Server Usage

Use the same endpoints from the server through `auth.api.*`.

```ts title="server.ts"
import { headers } from "next/headers"
import { auth } from "@/lib/auth"

const requestHeaders = await headers()
const challenge = await auth.api.getPendingTwoFactorChallenge({
    headers: requestHeaders,
})

const totpMethod = challenge.methods.find((method) => method.kind === "totp")

await auth.api.verifyTwoFactor({
    headers: requestHeaders,
    body: {
        methodId: totpMethod!.id,
        code: "123456",
    },
})
```

<Callout type="warn">
  If you are continuing a paused sign-in with `auth.api.*`, forward the relevant request headers so Better Auth can read and update the two-factor and session cookies.
</Callout>

## Options

### Server

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { twoFactor } from "better-auth/plugins"

export const auth = betterAuth({
    plugins: [
        twoFactor({
            issuer: "Acme",
            totpOptions: {
                digits: 6,
                period: 30,
            },
            otpOptions: {
                period: 3,
                digits: 6,
                sendOTP: async ({ user, otp }) => {
                    console.log(`Send ${otp} to ${user.email}`)
                },
            },
            recoveryCodeOptions: {
                amount: 10,
                length: 12,
            },
            trustDevice: {
                maxAge: 60 * 60 * 24 * 7,
                requireReverificationFor: ["/sign-in/email"],
            },
            enforcement: {
                decide({ primaryMethod, request }) {
                    if (primaryMethod === "company-sso") {
                        return { decision: "skip", reason: "idp-mfa" }
                    }

                    if (request?.headers.get("x-high-risk") === "true") {
                        return { decision: "enforce", reason: "high-risk-sign-in" }
                    }
                },
            },
            skipVerificationOnEnable: false,
            pendingChallengeMaxAge: 60 * 10,
            maxVerificationAttempts: 5,
        }),
    ],
})
```

**issuer**: The issuer used for `TOTP` provisioning URIs. By default it uses your app name.

**totpOptions**

* `digits`: Number of `TOTP` digits. Default `6`.
* `period`: `TOTP` period in seconds. Default `30`.
* `disable`: Disable `TOTP` enrollment.

**otpOptions**

* `sendOTP`: Function used to send the `OTP` to the user.
* `period`: `OTP` lifetime in minutes. Default `3`.
* `digits`: Number of `OTP` digits. Default `6`.
* `allowedAttempts`: Failed attempts allowed before Better Auth requires a new `OTP`.
* `storeOTP`: How `OTP` values are stored. Supports `"plain"`, `"encrypted"`, `"hashed"`, or custom handlers.

**recoveryCodeOptions**

* `amount`: Number of recovery codes to generate. Default `10`.
* `length`: Recovery code length. Default `12`.
* `customGenerate`: Custom recovery-code generator.

**skipVerificationOnEnable**: Mark newly created `TOTP` or `OTP` methods as verified immediately.

**pendingChallengeMaxAge**: Pending sign-in challenge lifetime in seconds. Default `600`.

**trustDevice**

* `maxAge`: Trusted-device lifetime in seconds. Default `604800` (7 days).
* `requireReverificationFor`: Exact sign-in paths that always require a fresh 2FA check, even on a trusted device.

**enforcement**

* `decide(input)`: Server-side hook that returns `"enforce"`, `"skip"`, `undefined`, or `{ decision, reason }` for a candidate challenge.

**maxVerificationAttempts**: Failed verification attempts allowed per paused sign-in before the attempt is locked. Default `5`.

**schema**: Override table names and field names for the plugin's tables.

import { createAuthClient } from "better-auth/client"

## Schema

The plugin adds four tables:

* `twoFactorMethod`: Stores method metadata such as `userId`, `kind`, `label`, `verifiedAt`, and `lastUsedAt`
* `twoFactorTotp`: Stores encrypted `TOTP` secrets
* `twoFactorRecoveryCode`: Stores hashed recovery codes
* `trustedDevice`: Stores trusted-device records

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { twoFactor } from "better-auth/plugins"

export const auth = betterAuth({
    plugins: [
        twoFactor({
            schema: {
                twoFactorMethod: {
                    modelName: "auth_methods",
                    fields: {
                        userId: "user_id",
                        verifiedAt: "verified_at",
                        lastUsedAt: "last_used_at",
                    },
                },
                twoFactorTotp: {
                    modelName: "auth_method_totp",
                    fields: {
                        methodId: "method_id",
                    },
                },
                twoFactorRecoveryCode: {
                    modelName: "auth_method_recovery_codes",
                    fields: {
                        methodId: "method_id",
                        codeHash: "code_hash",
                    },
                },
                trustedDevice: {
                    modelName: "auth_trusted_devices",
                    fields: {
                        userId: "user_id",
                        lookupKeyHash: "lookup_key_hash",
                        lastUsedAt: "last_used_at",
                        expiresAt: "expires_at",
                    },
                },
            },
        }),
    ],
})
```
