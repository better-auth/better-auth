Teams are a way to group members within an organization.

## Set Up 

<Steps>
    <Step>
        ### 1. Add the teams addon
       To add the teams addon, simply add the `teams()` addon into the `organization()` plugin.

        ```ts title="auth.ts"
        import { organization } from "@better-auth/organization";
        import { teams } from "@better-auth/organization/addons"; // [!code highlight]

        export const auth = betterAuth({
            plugins: [
                organization({
                    use: [teams()], // [!code highlight]
                }),
            ],
        });
        ```
    </Step>

    <Step>
        ### 2. Migrate the database
        Run the migration or generate the schema to add the necessary fields and tables to the database.

        <Tabs items={["migrate", "generate"]}>
            <Tab value="migrate">
                ```bash
                npx @better-auth/cli migrate
                ```
            </Tab>
            <Tab value="generate">
                ```bash
                npx @better-auth/cli generate
                ```
            </Tab>
        </Tabs>
    </Step>
</Steps>

That's it! You can now start utilizing team functionality in your application.

## Creating a Team

You can create a team within an organization using the `organization.createTeam` method.

<APIMethod
    path="/organization/create-team"
    method="POST"
    requireSession>
```ts
type createTeam = {
    /**
     * The name of the team.
     */
    name: string = "Engineering"
    /**
     * The organization ID which the team will belong to.
     * If not provided, uses the active organization.
     */
    organizationId?: string = "org-id"
    /**
     * The slug of the team.
     * Only required if `enableSlugs` is set to true in the teams addon options.
     */
    slug?: string = "engineering"
}
```
</APIMethod>

Returns the created team:

```ts
type Result = {
    id: string;
    name: string;
    organizationId: string;
    createdAt: Date;
    updatedAt: Date;
    // Additional fields if configured in schema
}
```

<Callout type="info">
    The user must have `team:create` permission to create a team. By default, this permission is granted to `owner` and `admin` roles.
</Callout>

<Callout type="warn">
    You can limit the maximum number of teams per organization using the `maximumTeams` option in the teams addon configuration.
</Callout>

## Removing a Team

You can remove a team from an organization using the `organization.removeTeam` method.

<APIMethod
    path="/organization/remove-team"
    method="POST"
    requireSession>
```ts
type removeTeam = {
    /**
     * The team ID of the team to remove.
     */
    teamId: string = "team-id"
    /**
     * The organization ID which the team falls under.
     * If not provided, uses the active organization.
     */
    organizationId?: string = "org-id"
}
```
</APIMethod>

Returns:

```ts
type Result = {
    message: "Team removed successfully."
}
```

<Callout type="warn">
    By default, the last team in an organization cannot be removed.
    To allow removing all teams, set `allowRemovingAllTeams: true` in the teams addon options.
</Callout>

## Listing Teams

You can list all teams in an organization using the `organization.listTeams` method. This endpoint supports pagination and sorting.

<APIMethod
    path="/organization/list-teams"
    method="GET"
    requireSession>
```ts
type listOrganizationTeams = {
    /**
     * The organization ID which the teams belong to.
     * If not provided, uses the active organization.
     */
    organizationId?: string = "org-id"
    /**
     * Maximum number of teams to return (1-100).
     */
    limit?: number = 10
    /**
     * Number of teams to skip for pagination.
     */
    offset?: number = 0
    /**
     * Field to sort by.
     * Options: "createdAt", "name", "updatedAt"
     */
    sortBy?: "createdAt" | "name" | "updatedAt" = "createdAt"
    /**
     * Sort direction.
     */
    sortDirection?: "asc" | "desc" = "desc"
}
```
</APIMethod>

Returns a paginated result with teams and total count:

```ts
type Result = {
    teams: Array<{
        id: string;
        name: string;
        organizationId: string;
        createdAt: Date;
        updatedAt: Date;
        // Additional fields if configured in schema
    }>;
    /**
     * Total count of teams in the organization.
     */
    total: number;
}
```

<Callout type="info">
    The user must be a member of the organization to list its teams.
</Callout>

You can get a specific team by its ID or slug using the `organization.getTeam` method.

<APIMethod
    path="/organization/get-team"
    method="GET"
    requireSession>
```ts

```

Returns the team:

```ts
type Result = {
    id: string;
    name: string;
    organizationId: string;
    createdAt: Date;
    updatedAt: Date;
    slug?: string; // If slugs are enabled
    // Additional fields if configured in schema
}
```

<Callout type="info">
    The user must be a member of the organization to get a team.
</Callout>

## Adding a Team Member

You can add an existing organization member to a team using the `organization.addTeamMember` method. The user must already be a member of the organization before they can be added to a team.

    requireSession

```ts
type addTeamMember = {

     * The team the user should be a member of.
     */
    teamId: string = "team-id"
    /**
     * The user ID which represents the user to be added as a member.
     */
    userId: string = "user-id"
    /**
     * The organization ID which the team falls under.
     * If not provided, uses the active organization.
     */
    organizationId?: string = "org-id"
}
```
</APIMethod>

Returns the created team member:

```ts
type Result = {
    id: string;
    userId: string;
    teamId: string;
    createdAt: Date;
    // Additional fields if configured in schema
}
```

<Callout type="info">
    The user must have `member:update` permission to add members to a team. By default, this permission is granted to `owner` and `admin` roles.
</Callout>

<Callout type="warn">
    You can limit the maximum number of members per team using the `maximumMembersPerTeam` option in the teams addon configuration.
</Callout>

## Setting the Active Team

You can set a team as the active team for the current session using the `organization.setActiveTeam` method. This stores the active team ID in the session, making it available for subsequent API calls.

<APIMethod
    path="/organization/set-active-team"
    method="POST"
    requireSession>
```ts
type setActiveTeam = {
    /**
     * The team ID to set as active.
     * Pass null to unset the active team.
     */
    teamId?: string | null = "team-id"
}
```
</APIMethod>

Returns the team that was set as active, or `null` if the active team was unset:

```ts
type Result = {
    id: string;
    name: string;
    organizationId: string;
    createdAt: Date;
    updatedAt: Date;
    // Additional fields if configured in schema
} | null
```

<Callout type="info">
    The user must be a member of the team to set it as active. An active organization must also be set before setting an active team.
</Callout>

## Teams Addon Options

You can configure the teams addon with the following options:

### `defaultTeam`

Configure the default team that is automatically created when a new organization is created.

```ts
teams({
    defaultTeam: {
        enabled: true, // Enable/disable default team creation
        customCreateDefaultTeam: async (organization) => {
            return {
                name: `${organization.name} Team`,
                // Additional fields
            };
        }
    }
})
```

**Default:** `{ enabled: true }`

### `maximumTeams`

The maximum number of teams allowed per organization. Can be a number or a function.

```ts
teams({
    maximumTeams: 10
})

// Or with a function
teams({
    maximumTeams: async ({ organizationId, session }) => {
        // Return different limits based on organization or user
        return 10;
    }
})
```

**Default:** `Infinity`

### `maximumMembersPerTeam`

The maximum number of members allowed per team. Can be a number or a function.

```ts
teams({
    maximumMembersPerTeam: 50
})

// Or with a function
teams({
    maximumMembersPerTeam: async ({ teamId, organizationId, session }) => {
        return 50;
    }
})
```

**Default:** `Infinity`

### `allowRemovingAllTeams`

By default, the last team in an organization cannot be removed. Set this to `true` to allow removing all teams.

```ts
teams({
    allowRemovingAllTeams: true
})
```

**Default:** `false`

### `enableSlugs`

Enable slugs for teams. When enabled, teams can be identified by their slug in addition to their ID.

```ts
teams({
    enableSlugs: true
})
```

**Default:** `false`

### `defaultTeamIdField`

Controls the input & output of team identifiers. If set to `slug`, endpoint parameters for `teamId` will accept the team's `slug` instead of its `id`.

```ts
teams({
    defaultTeamIdField: "slug" // "slug" or "id"
})
```

**Default:** `"id"`

### `schema`

Customize the database schema for team-related tables. You can rename fields, change model names, or add additional fields.

```ts
teams({
    schema: {
        team: {
            modelName: "project", // Rename the table
            fields: {
                name: "projectName" // Rename fields
            },
            additionalFields: {
                description: {
                    type: "string",
                    required: false
                },
                isArchived: {
                    type: "boolean",
                    required: false,
                    defaultValue: false
                }
            }
        },
        teamMember: {
            additionalFields: {
                department: {
                    type: "string",
                    required: false
                },
                isLead: {
                    type: "boolean",
                    required: false
                }
            }
        }
    }
})
```

Additional fields will be available in:
- `createTeam` / `updateTeam` request body (for team fields)
- `addTeamMember` / `updateTeamMember` request body (for teamMember fields)
- All response types that return team or team member data

<Callout type="warn">
    Don't forget to re-run schema generation or migrations after changing schema options.
</Callout>

### `hooks`

Hooks allow you to run custom logic at various points in the team lifecycle.

```ts
teams({
    hooks: {
        beforeCreateTeam: async ({ team, user, organization }, ctx) => {
            // Runs before a team is created
            // Return { data: {...} } to modify the team data
        },
        afterCreateTeam: async ({ team, user, organization }, ctx) => {
            // Runs after a team is created
        },
        beforeUpdateTeam: async ({ team, updates, user, organization }, ctx) => {
            // Runs before a team is updated
            // Return { data: {...} } to modify the update data
        },
        afterUpdateTeam: async ({ team, user, organization }, ctx) => {
            // Runs after a team is updated
        },
        beforeDeleteTeam: async ({ team, user, organization }, ctx) => {
            // Runs before a team is deleted
        },
        afterDeleteTeam: async ({ team, user, organization }, ctx) => {
            // Runs after a team is deleted
        },
        beforeAddTeamMember: async ({ teamMember, team, user, organization }, ctx) => {
            // Runs before a member is added to a team
            // Return { data: {...} } to modify the team member data
        },
        afterAddTeamMember: async ({ teamMember, team, user, organization }, ctx) => {
            // Runs after a member is added to a team
        },
        beforeUpdateTeamMember: async ({ teamMember, updates, team, user, organization }, ctx) => {
            // Runs before a team member is updated
            // Return { data: {...} } to modify the update data
        },
        afterUpdateTeamMember: async ({ teamMember, team, user, organization }, ctx) => {
            // Runs after a team member is updated
        },
        beforeRemoveTeamMember: async ({ teamMember, team, user, organization }, ctx) => {
            // Runs before a member is removed from a team
        },
        afterRemoveTeamMember: async ({ teamMember, team, user, organization }, ctx) => {
            // Runs after a member is removed from a team
        }
    }
})
```

## Team Schema

The team table includes the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier |
| `name` | `string` | Team name |
| `organizationId` | `string` | Reference to the organization |
| `slug` | `string?` | URL-friendly identifier (if `enableSlugs` is true) |
| `createdAt` | `Date` | Creation timestamp |
| `updatedAt` | `Date?` | Last update timestamp |

## Team Member Schema

The teamMember table includes the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier |
| `teamId` | `string` | Reference to the team |
| `userId` | `string` | Reference to the user |
| `createdAt` | `Date?` | Creation timestamp |
