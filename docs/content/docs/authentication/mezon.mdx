<Steps>
    <Step> 
        ### Get your Mezon credentials
        To use Mezon sign in, you need a client ID and client secret. You can get them from the [Mezon Developer Portal](https://mezon.ai/developers).

        Make sure to set the redirect URL to `http://localhost:3000/api/auth/callback/mezon` for local development. For production, you should set it to the URL of your application. If you change the base path of the auth routes, you should update the redirect URL accordingly.
    </Step>

  <Step>
        ### Configure the provider
        To configure the provider, you need to import the provider and pass it to the `socialProviders` option of the auth instance.

        ```ts title="auth.ts"
        import { betterAuth } from "better-auth"
        
        export const auth = betterAuth({
            socialProviders: {
                mezon: { // [!code highlight]
                    clientId: process.env.MEZON_CLIENT_ID as string, // [!code highlight]
                    clientSecret: process.env.MEZON_CLIENT_SECRET as string, // [!code highlight]
                }, // [!code highlight]
            },
        })
        ```
    </Step>
       <Step>
        ### Sign In with Mezon
        To sign in with Mezon, you can use the `signIn.social` function provided by the client. The `signIn` function takes an object with the following properties:
        - `provider`: The provider to use. It should be set to `mezon`.

        ```ts title="auth-client.ts"  
        import { createAuthClient } from "better-auth/client"
        const authClient =  createAuthClient()
        
        const signIn = async () => {
            const data = await authClient.signIn.social({
                provider: "mezon"
            })
        }
        ```
    </Step>
</Steps>

## Usage

### Setting up your Mezon app

Mezon has two types of apps: Mezon apps and Mezon Bots.
For Mezon apps, you only use app for authentication or working with channel applications. 
For Mezon Bots, you can use the bot token for authentication or develop your own bot applications

### Why don't I have a refresh token?

Mezon doesn't issue refresh tokens for OAuth apps. For regular OAuth apps,
There's no need for a refresh token because the access token doesn't expire on a short interval like Google or Discord.
