title: SolidStart Integration
description: Integrate Better Auth with SolidStart.
---

Before you start, make sure you have a Better Auth instance configured. If you haven't done that yet, check out the [installation](/docs/installation).

### Mount the handler

We need to mount the handler to SolidStart server. Put the following code in your `[...auth].ts` file inside `/routes/api/auth` folder.

```ts title="[...auth].ts"
import { auth } from "~/lib/auth";
import { toSolidStartHandler } from "better-auth/solid-start";

export const { GET, POST } = toSolidStartHandler(auth);
```

## Create a client

Create a client instance using `better-auth/solid`:

```ts title="lib/auth-client.ts"
import { createAuthClient } from "better-auth/solid";

export const authClient = createAuthClient({
  // you can pass client configuration here
});
```

## Using the Auth Component

Better Auth provides a pre-built `<Auth />` component for quick authentication UI:

```tsx title="routes/login.tsx"
import { authClient } from "~/lib/auth-client";
import { Auth } from "better-auth/solid";
import { useNavigate } from "@solidjs/router";

export default function LoginPage() {
  const navigate = useNavigate();
  
  return (
    <div class="container">
      <Auth
        ui={authClient.ui.signIn()}
        onSuccess={(data) => {
          navigate(data.redirectTo || "/dashboard");
        }}
        onError={(err) => console.error(err.message)}
      />
    </div>
  );
}
```

The component automatically inherits your shadcn theme if you're using one. For more customization options, see the [UI Pages documentation](/docs/concepts/ui).

## Use the session

Use `authClient.useSession()` to reactively access the current session:

```tsx
import { authClient } from "~/lib/auth-client";

export default function Profile() {
  const session = authClient.useSession();
  
  return (
    <div>
      {session().data ? (
        <p>Welcome, {session().data.user.name}</p>
      ) : (
        <p>Not signed in</p>
      )}
    </div>
  );
}
```
